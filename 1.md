## What This Platform Does

Customers place parcel delivery orders. Drivers (two-wheelers, three-wheelers, Tata Ace) receive and fulfill those orders. An admin panel controls KYC, driver management, and operational monitoring.

The core loop:

```
Customer pays â†’ Order dispatched to drivers via FCM â†’ Driver accepts â†’ Pickup â†’ Delivery
```

If no driver accepts within 10 minutes, the order expires and the customer is automatically refunded.

## Apps

### Customer App (`apps/customer/`)
Flutter mobile app for Android and iOS. Customers:
- Authenticate via Firebase phone OTP or Google Sign-In
- Create delivery orders (pickup address, drop address, vehicle type)
- Pay via Razorpay mobile SDK
- Track order status in real time (polling)
- Cancel orders and receive automatic refunds
- Rate completed deliveries

### Driver App (`apps/driver/`)
Flutter mobile app. Drivers:
- Authenticate via Firebase phone OTP
- Toggle online/offline
- Pay a daily platform fee via Razorpay before receiving orders
- See incoming orders via FCM notifications + polling `/driver/orders/incoming`
- Accept, pick up, transit, and complete deliveries
- Track earnings and payout history

### Admin Panel (`apps/admin/`)
Static HTML + vanilla JS. Served from any static host or opened locally. Connects to the backend API using an admin JWT. Capabilities:
- View and approve/reject driver KYC documents
- Monitor live orders and driver status
- Force-cancel orders
- View customer list
- Block/unblock users
- Manage driver subscriptions and platform fees

## Backend (`backend/`)

NestJS application, single process, deployed on Railway. Connects to:
- **PostgreSQL** (hosted on Supabase) via Prisma ORM
- **Firebase Admin SDK** for ID token verification and FCM push notifications
- **Razorpay API** for payment order creation and refunds
- **Supabase Storage** for driver KYC document uploads

| Module | Responsibility |

| `payments` | Razorpay order creation, HMAC verification, `finalizePayment`, refunds |

## Core Data Flow

### Order Placement
1. Customer calls `POST /orders` â†’ order created with `status=PENDING`
2. Backend returns Razorpay order ID to Flutter SDK
3. Customer completes payment in Flutter (Razorpay SDK)
4. Flutter calls `POST /payments/verify` with HMAC signature
5. Backend verifies signature, updates order to `status=CREATED`, `paymentStatus=paid`, sets `expiresAt = now + 10min`
6. Backend emits `order.created` event â†’ `DispatchService` picks it up asynchronously

### Dispatch
1. `DispatchService.notifyEligibleDrivers()` queries online, KYC-approved, city+vehicle-matched, heartbeat-active drivers
2. Sends FCM push to each eligible driver
3. Drivers also poll `GET /driver/orders/incoming` every ~4 seconds
4. `retryDispatch` cron runs every 2 minutes for unaccepted CREATED orders still within TTL
5. `expireCreatedOrders` cron cancels and refunds orders past `expiresAt`

### Delivery
1. Driver calls `POST /driver/orders/:id/accept` â†’ atomic assignment
2. Driver calls `/pickup` â†’ `PICKED_UP`
3. Driver calls `/transit` â†’ `IN_TRANSIT`
4. Driver calls `/complete` â†’ `DELIVERED`
5. FCM notification sent to customer at each step

## Authentication Model

| Actor | Auth Method | Token |
|-------|-------------|-------|
| Customer | Firebase phone OTP / Google Sign-In â†’ `POST /customer/auth/firebase` | JWT (7d) |
| Driver | Firebase phone OTP â†’ `POST /driver/auth/firebase` (or dev-login in dev) | JWT (7d) |
| Admin | Email+password â†’ `POST /admin/auth/login` | JWT |

The JWT payload includes either `userId` (customer) or `driverId` (driver). Role separation is enforced by checking which field is present on the JWT payload.

## Environment Separation

There is currently **one Railway deployment** (`dashdelivery-dev` branch). There is no separate production environment yet. Before public launch, a separate production Railway service with isolated environment variables must be provisioned.

Dev-only features that must be disabled in production:
- `POST /payments/dev-confirm` â€” bypasses Razorpay payment
- `POST /customer/auth/dev-login` â€” bypasses Firebase auth
- `POST /driver/auth/dev-login` â€” bypasses Firebase auth
- `RAZORPAY_MOCK=true` env var â€” bypasses HMAC signature verification
